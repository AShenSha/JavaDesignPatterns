命令模式的优点
    1.降低了系统的耦合度.由于请求者和接收者之间不存在直接引用,因此两者之间能现实完全解耦.
    2.新的命令可以很容易的加入到系统中.由于增加新的类不会影响到其他类
    3.可以比较容易的设计一个命令队列或者宏命令(组合命令)
    4.为请求的撤销和恢复操作提供了一种设计和方案
缺点
    1.使用命令模式会使系统中有过多的具体命令类,
命令模式的适用场景
    1.系统需要将请求调用者和接收者解耦,使得调用者和接收者不直接交互
    2.系统需要在不同的时间指定请求,将请求排队和执行请求
    3.系统需要支持命令的撤销和恢复操作
    4.系统需要将一组操作组合在一起行程宏命令
